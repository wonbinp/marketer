메이아이 마케팅 콘텐츠(보도자료 또는 블로그)를 작성하는 커맨드입니다.

## 진행 순서

### 1단계: 콘텐츠 유형 및 정보 파악

사용자에게 아래 정보를 **한 번에 하나씩 순서대로** 확인합니다. 사용자가 이미 제공한 정보는 다시 묻지 않고 건너뜁니다.

> **UI 규칙**: 선택지가 4개를 초과하는 질문(예: 보도자료 유형 7종)은 선택지 UI 대신 텍스트로 목록을 나열하고 사용자가 직접 입력하도록 한다. 사용자에게 보여주는 텍스트에는 마크다운 서식(`**`, `*`, `#` 등)을 사용하지 않는다.

#### 질문 순서

**① 콘텐츠 유형**: 보도자료 / 블로그

**보도자료인 경우:**
- **② 유형**: 투자유치 / 수상·논문 / 실적·성과 / 고객사 도입 / 제품 출시 / 정부 지원 / 인터뷰
- **③ 핵심 정보**: 보도자료에 포함할 사실관계 (예: 투자 규모, 수상명, 고객사명, 매장 수 등)
- **④ 인용문**: 대표/관계자 코멘트가 있으면 제공받기 (없으면 "없음"으로 건너뛰기)
- **⑤ 희망 배포일/엠바고**: 희망 배포일과 엠바고 여부를 함께 확인 (배포일 미지정 시 오늘로부터 4주 후 가장 가까운 화~목요일을 기본값으로 제안하되, 그 이유도 함께 안내 (예: "기본값은 YYYY년 MM월 DD일(X요일)입니다. 고객사 홍보팀 검토 기간 약 4주를 고려했으며, 기사화에 유리한 화~목요일로 설정했습니다."), 엠바고 불필요 시 "없음")
- **⑥ 메이아이 문의처**: 보도자료 하단에 넣을 메이아이 측 담당자명, 이메일, 전화번호 (없으면 "없음"으로 건너뛰기)

**블로그인 경우:**
- **② 카테고리**: 고객 사례 / 제품 소개 / 인사이트
- **③ 주제/소재**: 글의 핵심 주제
- **④ 타겟 독자**: 어떤 업종/직무의 독자를 대상으로 하는지
- **⑤ 포함할 정보**: 구체적 수치, 사례, 기능명 등

---

### 2단계: 스타일 가이드 읽기

콘텐츠 유형에 맞는 스타일 가이드를 읽습니다.

**보도자료** (출력 양식도 함께 읽기):
- `styles/press-release-format.md` — 보도자료 출력 양식 (필수)
- 유형별 스타일 가이드:
- `styles/press-release-investment.md` — 투자 유치
- `styles/press-release-awards.md` — 수상/논문 채택
- `styles/press-release-performance.md` — 실적/성과
- `styles/press-release-customers.md` — 고객사 도입
- `styles/press-release-products.md` — 제품 출시/업데이트
- `styles/press-release-government.md` — 정부 지원 사업 선정
- `styles/press-release-interviews.md` — 인터뷰/기업 프로필

**블로그:**
- `styles/blog-customers.md` — 고객 사례
- `styles/blog-products.md` — 제품 소개
- `styles/blog-insights.md` — 인사이트/트렌드 분석

유형이 여러 개 섞인 경우, 관련 스타일 가이드를 모두 읽되 주된 유형을 기본으로 합니다.

---

### 3단계: 원문 참고

해당 유형의 원문을 최신 기준으로 최대 10편까지 읽어 실제 톤과 표현 수준을 파악합니다.

- **보도자료**: `data/articles/{유형}/` 폴더에서 최신 기사부터 최대 10편
- **블로그**: `data/blog/{카테고리}/` 폴더에서 최신 글부터 최대 10편

원문의 구조, 문장 길이, 단락 구성을 참고합니다.

---

### 4단계: 초안 작성

스타일 가이드의 체크리스트를 따라 콘텐츠를 작성합니다.

#### 보도자료 작성 규칙
- **출력 양식**: `styles/press-release-format.md`의 양식을 따라 배포 정보 → 제목 → 본문 → 보일러플레이트 → 문의처 순서로 출력
- **리드**: 스타일 가이드의 리드 공식에 따라 핵심 사실을 첫 문장에 배치
- **본문**: 유형별 전형적 구성 순서를 따름
- **인용문**: 보도자료 말미에 배치, 직접 인용 + 간접 서술 혼합
- **문체**: `~했다` 서술체, 객관적 보도 톤
- **길이**: 스타일 가이드에 명시된 글자 수/문단 수 범위 내
- **보일러플레이트/문의처**: `press-release-format.md`에 정의된 정형 문구를 하단에 자동 포함. 보일러플레이트의 고객사는 `press-release-format.md`의 고객사 노출 규칙(전체 목록 제시 → 사용자 확인 → 대표 3~4곳 선정)을 따른다.

#### 블로그 작성 규칙
- **구조**: 제목 → 부제 → 도입부(문제 제기) → 본문(소제목 3~6개) → 마무리(CTA)
- **문체**: `~입니다/~습니다` 기본 + `~인데요/~거예요` 혼용으로 친근함 유지
- **톤**: 매쉬를 도입부터 전면에 내세우지 않고, 문제 공감 → 한계 → 매쉬 해결 순서로 연결
- **CTA**: 마지막에 소개서 받기 또는 contact@may-i.io 안내
- **길이**: 일반 글 1,000~2,000자, 심화 글 2,000~3,500자

---

### 5단계: 제목 작성

- **보도자료**: 스타일 가이드의 제목 패턴을 참고하여 제목을 2~3개 제안
- **블로그**: 질문형/선언형/방법 제시형 중 적합한 형태로 제목 + 부제 제안

사용자가 제목을 선택하거나 확정한 뒤 다음 단계로 넘어갑니다.

---

### 6단계: 보일러플레이트 고객사 확인 (보도자료만 해당)

`press-release-format.md`의 고객사 노출 규칙에 따라 전체 고객사 목록을 사용자에게 보여주고, 현재 고객사가 아닌 곳을 제거한 뒤 대표 3~4곳을 선정합니다.

---

### 7단계: 검수

스타일 가이드의 체크리스트 항목을 하나씩 확인하고, 누락된 요소가 있으면 보완합니다.

---

### 8단계: 파일 저장

보도자료는 MS Word(.docx) 파일로 저장합니다. `python-docx` 라이브러리를 사용합니다.

- **저장 경로**: `output/[메이아이] {제목 요약} YYMMDD.docx`
- **파일명 규칙**: `[메이아이] {제목을 간결하게 요약} YYMMDD.docx` (YYMMDD는 배포일 기준)
- **파일명 예시**: `[메이아이] 코웨이 매쉬 확장 도입 계약 체결 260413.docx`
- **레이아웃**: `styles/press-release-format.md`의 docx 스타일 규칙을 따라 1열 3행 테이블로 생성 (1행: 배포 정보/회색 배경, 2행: 제목 및 내용/흰 배경, 3행: 회사 소개+문의/회색 배경)

저장 완료 후 파일 경로를 사용자에게 안내합니다.

## 주의사항
- 사용자가 제공하지 않은 사실(숫자, 고객사명, 날짜 등)을 임의로 넣지 않습니다.
- 추가 정보를 요청할 때 "꼭 필요한 정보"라는 표현 대신 "있으면 더 좋은 정보"로 안내합니다. 사용자가 제공한 정보만으로도 작성은 가능하되, 추가 정보가 있으면 더 풍성한 보도자료가 된다는 뉘앙스를 유지합니다.
- 영문명은 첫 등장 시 괄호로 병기합니다: `매쉬(mAsh)`, `핵심성과지표(KPI)` 등
- **보도자료와 블로그는 문체가 다릅니다.** 보도자료는 `~했다` 서술체, 블로그는 `~입니다/~해요` 혼용체. 절대 섞지 않습니다.
