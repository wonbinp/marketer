# Marketer - 메이아이 콘텐츠 생성 봇

## 프로젝트 비전
- **단기 목표**: 메이아이(mAy-I)의 보도자료와 블로그 글을 기존 스타일과 일관되게 작성하는 콘텐츠 생성 도구.
- **장기 목표**: 마케터를 채용하지 않고 이 봇으로 마케팅 콘텐츠 업무를 대체. 보도자료·블로그 작성뿐 아니라 마케팅 전반의 기획·실행을 지원하는 방향으로 확장.

## 디렉토리 구조
```
marketer/
├── data/
│   ├── articles/          ← 보도 기사 원문 — 참고자료 (유형별 폴더)
│   │   ├── investment/    ← 투자 유치
│   │   ├── awards/        ← 수상/논문 채택
│   │   ├── performance/   ← 실적/성과
│   │   ├── customers/     ← 고객사 도입
│   │   ├── products/      ← 제품 출시/업데이트
│   │   ├── government/    ← 정부 지원 사업 선정
│   │   └── interviews/    ← 인터뷰/기업 프로필
│   └── blog/              ← 블로그 원문 (카테고리별 폴더)
│       ├── customers/     ← 고객 사례
│       ├── products/      ← 제품 소개
│       └── insights/      ← 인사이트/트렌드 분석
├── styles/                ← 유형별 스타일 가이드
│   ├── press-release-*.md ← 보도자료 스타일 (7종)
│   └── blog-*.md          ← 블로그 스타일 (3종)
└── .claude/commands/      ← 슬래시 커맨드
```

## 콘텐츠 작성 트리거

사용자가 보도자료, 기사, 블로그, 콘텐츠 등의 **작성/제작을 요청**하면(슬래시 커맨드 없이도), `.claude/commands/write-content.md`의 워크플로우를 읽고 동일하게 따른다.
- 예: "보도자료 써줘", "블로그 하나 작성해줘", "투자 유치 콘텐츠 만들어줘", "고객 사례 글 쓸 건데" 등

## 콘텐츠 작성 시 공통 규칙

### 회사 정보
- 회사명: 메이아이 / mAy-I / (주)메이아이
- 솔루션명: 매쉬(mAsh) — 영상처리 AI 기반 오프라인 방문객 데이터 분석 솔루션
- 대시보드: 매쉬보드(mAsh Board)
- 블로그: https://blog.may-i.io/

### 작성 원칙
1. **스타일 가이드를 반드시 먼저 읽는다.** 해당 유형의 `styles/` 파일을 읽고 문체, 구조, 표현을 따른다.
2. **원문을 참고한다.** `data/` 폴더의 원문 2~3편을 few-shot 예시로 읽어 톤과 수준을 맞춘다.
3. **유형이 섞인 경우** 관련 스타일 가이드를 모두 참고하되, 주된 유형의 스타일을 기본으로 한다.
4. **보도자료와 블로그는 문체가 다르다.** 보도자료는 `~했다` 서술체, 블로그는 `~입니다/~해요` 혼용체.
5. **과장하지 않는다.** 사용자가 제공한 팩트만으로 작성하고, 없는 정보를 지어내지 않는다.
