# 블로그 인사이트 글 스타일 가이드

> 이 문서는 메이아이 블로그 인사이트 카테고리 기존 글 7편을 분석하여 도출한 스타일 가이드입니다.
> 글 생성 시 시스템 프롬프트에 포함하여 일관된 톤과 구조를 유지하는 데 사용합니다.

---

## 1. 문체

### 어미 패턴: ~입니다/~해요 혼용체

- 기본 어미는 **~해요/~이에요/~고요** (해요체)를 주로 사용한다.
- 핵심 사실, 정의, 수치를 전달할 때는 **~입니다/~했습니다** (합니다체)로 전환하여 신뢰감을 준다.
- 하나의 단락 안에서도 해요체와 합니다체를 자연스럽게 섞어 쓴다.
- **절대 사용하지 않는 것:** ~이다/~한다 (이다체, 저널리즘 톤). 단, 시암 피왓 글은 예외적으로 이다체를 사용했으나, 이 글을 제외하면 전체적으로 해요/합니다 혼용체가 표준이다.

### 실제 예시

```
해요체 (일상적 서술):
- "코로나19 시기도 무사히 넘기고 성과가 하락세로 바뀐 적이 한 번도 없던 거예요."
- "직원도 38만 명 정도 근무하고 있고요."
- "리테일 AI를 도입해 미래 성장 동력을 마련하고 있고요."
- "이런 일이 빈번하게 발생해 큰 골칫거리였습니다."
- "이마트 가면 결제할 때 포인트 적립하는 거랑 비슷한데요."

합니다체 (핵심 팩트, 성과):
- "1989년 첫 번째 매장을 연 뒤 35년 연속으로 매출과 영업이익이 늘어왔습니다."
- "발주 업무에 드는 시간을 최대 40% 단축했다고 밝혔습니다."
- "2024년에 달성한 매출은 약 961억 유로, 원화로 약 155조 원이에요."
```

### 격식 수준과 친근함

- **대화하듯 설명하되, 전문 지식을 갖춘 톤**을 유지한다.
- 독자에게 직접 말을 건네는 표현을 자주 사용한다: "~잖아요", "~거든요", "~인데요", "~거예요", "~하고요"
- 가벼운 비유나 일상적 표현을 섞어 접근성을 높인다:
  - "어디나 비슷해서", "헛다리를 제대로 짚는 것입니다"
  - "산더미처럼 쌓인 저가 제품"
  - "응대하다가 퇴근하는 건 과언이 아니겠고요"

## 2. 글 전체 구조

### 표준 흐름: 도입 → 기업 소개 → 사례 나열 → 자사 연결(CTA)

모든 글은 아래 4단계 구조를 따른다:

| 순서 | 섹션 | 역할 |
|------|------|------|
| 1 | **도입부** | 인사 + 흥미 유발 질문/상황 제시 + 글의 방향 예고 |
| 2 | **기업/시장 배경** | 해당 기업 또는 시장의 규모·역사·특징 소개 |
| 3 | **AI 활용 사례 (2~4개)** | 번호를 매기며 사례를 하나씩 깊이 있게 설명 |
| 4 | **자사 제품 연결 + CTA** | 사례 교훈 요약 → 매쉬(mAsh) 자연스럽게 연결 → 행동 유도 |

### 섹션 간 구분

- 각 주요 섹션 사이에 수평선(`---`)을 삽입한다.
- H2(`##`)로 대단원을 나누고, 필요 시 H3(`###`)로 세부 항목을 나눈다.

---

## 3. 제목 작성법

### 패턴

제목은 다음 두 가지 패턴 중 하나를 따른다:

**패턴 A: [기업/키워드] + [구체적 내용] + [숫자 or 의문문]**
```
- "일본 기념품 성지 돈키호테의 리테일 AI 활용 전략 3가지"
- "AI 리테일 테크 투자 ROI, 영국 테스코는 이렇게 달성했다"
- "리테일 매장 AI, 100년 기업 REWE는 어떻게 활용하고 있을까?"
```

**패턴 B: [주제] + [의문문/행동 유도]**
```
- "리테일 AI로 오프라인 매장 혁신하기, 어디서부터 시작할까?"
- "4가지로 정리하는 2025 상반기 AI 이슈"
- "월마트가 방문객 데이터로 매출을 끌어올린 비결: 리테일 미디어"
```

### 제목 규칙

- 숫자(3가지, 4가지, 100년 등)를 자주 포함한다.
- 의문문("~어떻게?", "~어디서부터?")이나 선언문("~이렇게 달성했다")을 혼용한다.
- 해외 기업명을 직접 노출하여 사례 기반임을 드러낸다.
- "리테일 AI", "AI 리테일 테크" 등 핵심 키워드를 포함한다.

### 부제목(H1 아래 볼드 텍스트 또는 일반 텍스트)

제목 바로 아래에 한 줄 부제목을 붙인다. 본문 내용을 간결하게 요약하거나, 글의 핵심 가치를 전달한다.
```
- "인공지능과 직원 경험의 시너지를 이뤄낸 비결"
- "물류부터 멤버십까지 테스코는 리테일 전 단계에 도입했어요"
- "운영 비용은 줄이고 고객 경험은 끌어올려보세요"
- "독일 2위 리테일 기업 REWE의 사례를 소개합니다"
- "딥시크 쇼크부터 AI 수석 임명까지"
```

## 4. 도입부 패턴

### 구성: 인사 → 흥미 유발 → 기업/주제 소개 → 본문 예고

도입부는 보통 3~5문장으로 이루어지며, 아래 순서를 따른다:

1. **인사:** `안녕하세요, 메이아이의 Content Editor 최창근입니다.` 또는 `안녕하세요, 오프라인 공간을 더 똑똑하게 만드는 인공지능 스타트업 메이아이입니다.`
2. **흥미 유발 질문 또는 친숙한 상황 제시:**
   - "한국에 홈플러스를 처음 탄생시킨 회사 어디일까요?"
   - "일본 여행 필수 코스로 꼽히는 이곳 어디일까요?"
   - "오프라인 리테일 매장은 둘로 나뉩니다. 리테일 AI를 도입해 매장 혁신에 앞장서는 곳과 아직도 도입하지 못해 끙끙 대는 곳."
3. **해당 기업/주제를 한국 독자가 알 만한 맥락으로 연결:**
   - "바로 영국에서 가장 큰 리테일 기업 테스코(Tesco)입니다. 1999년 삼성물산과 함께 홈플러스를 만들어 한국 시장에 진출했어요."
   - "가성비 좋은 기념품과 생필품을 사기 위해 들르는 곳으로 여겨지지만, 알고 보면 이미 리테일 AI를 도입해..."
4. **본문 예고 문장:**
   - "돈키호테는 리테일 AI로 어떻게 매장 운영 전략을 개선해나가고 있을까요?"
   - "이번 콘텐츠에서는 테스코가 리테일 AI를 어떻게 활용하는지 소개하겠습니다."
   - "REWE는 리테일 AI를 어느 부분에 어떻게 활용하고 있을까요? 두 가지 사례를 소개합니다."

### 주의사항

- 한국 독자가 공감할 수 있는 접점(한국 매장, 여행 경험 등)으로 시작한다.
- 통계나 시장 데이터로 시작하는 방식도 사용한다 (예: "리테일 업계 5곳 중 4곳이 'AI 도입' 전략을 세웠습니다").

## 5. 해외 사례 소개법

### 기업 소개 구조

해외 기업을 소개할 때 반드시 아래 요소를 포함한다:

1. **기업의 규모감:** 업계 순위, 매출 수치, 매장 수, 직원 수 등
2. **역사/성장 스토리:** 설립 연도, 성장 궤적, 연속 성장 기록 등
3. **경영 철학 또는 차별화 포인트:** 해당 기업만의 독특한 전략
4. **AI 도입 배경(동기):** 왜 AI를 도입하게 되었는지

### 실제 예시: 기업 규모감 표현

```
- "일본에서 이온몰과 함께 다섯 손가락 안에 꼽힐 정도로 몸집이 큰 유통기업이에요(업계 4위)."
- "2024년에 달성한 매출은 약 961억 유로, 원화로 약 155조 원이에요."
- "전 세계에 약 4500개, 그중에서 3700개를 영국에서 운영합니다."
- "일본 전역에 700여 개 매장을 운영 중이고"
```

### 사례 나열 방식

- 사례를 **"첫 번째", "두 번째", "세 번째"** 등 명시적 번호로 구분한다.
- 소제목에 번호를 포함한다: `## 리테일 AI 첫 번째, AI 가격 책정 시스템`
- 각 사례는 **문제 상황 → AI 솔루션 → 성과 수치** 순으로 서술한다.

## 6. 자주 쓰는 표현/관용구

### 연결 및 전환 표현

```
- "~인데요." (설명을 이어갈 때)
- "~하고요." / "~이고요." / "~있고요." (부연 설명을 덧붙일 때)
- "~거든요." (이유를 설명할 때)
- "~잖아요." (독자와 공감대를 형성할 때)
- "~거예요." / "~인 거죠." (핵심을 정리할 때)
- "~인 셈입니다." (결론을 도출할 때)
- "~고요." (문장 말미에 정보를 추가할 때 - 매우 빈번)
```

### 반복 등장하는 핵심 표현

```
문제 제기:
- "한계점도 분명했습니다"
- "큰 골칫거리였습니다"
- "감에만 의존해", "감을 기반으로"
- "경험에만 의존해"
- "~하기에는 오류가 생기는 부분이 많았습니다"
- "이커머스와 경쟁에서 살아남기 위해서"

해결/성과:
- "이러한 한계를 극복하기 위해"
- "이런 문제를 해결하기 위해"
- "~하는 성과를 거뒀습니다"
- "비용은 줄이고 매출은 늘리는"
- "운영 비용은 줄이고 고객 경험은 끌어올리"
- "데이터 기반 의사결정"
- "데이터 기반으로 전략을 세우고"
- "매출 상승에 기여합니다"
- "효율화", "효율적인"

자사 제품 연결:
- "매장 방문객 분석 AI"
- "오프라인 매장 방문객 솔루션 매쉬(mAsh)"
- "리테일 AI가 그 첫걸음이 될 수 있습니다"
- "매장 분석 AI 솔루션"
- "오프라인 공간을 똑똑하게 만드는"
```

### ~고요 패턴 (가장 특징적)

문장 끝에 "~고요."를 붙여 부연 설명을 가볍게 이어가는 패턴이 매우 빈번하다. 한 글에 5회 이상 등장하는 경우가 많다.
```
- "매출과 영업이익이 성장하는 데도 기여했고요."
- "현재는 5개 지점까지 확대된 상황이고요."
- "직원도 38만 명 정도 근무하고 있고요."
- "매장 직원이 그간 체감한 인사이트를 더하면 더 탄탄한 전략을 세울 수 있고요."
```

## 7. 데이터/수치 활용법

### 수치 제시 규칙

- 수치는 **볼드 처리**하여 시각적으로 강조한다.
- 한국 독자를 위해 원화 환산이나 비교 대상을 함께 제시한다.
- 구체적이고 인상적인 숫자를 선호한다.

### 실제 예시

```
환산/비교:
- "약 961억 유로, 원화로 약 155조 원"
- "주가 하락분을 한국 돈으로 따지면 약 863조 원, 우리 정부의 2024년 예산보다 1.27배 많았습니다."

성과 수치 (볼드):
- "직원이 가격 책정에 들이는 시간을 평균 **18시간에서 4.4시간**으로 줄이는 성과"
- "발주 업무에 드는 시간을 최대 **40% 단축**"
- "AI 도입 이후 클릭률이 **175% 증가**"
- "참여율 76%, 완주율 62%를 달성"
- "미국 마케터의 **46%**가 월마트에 광고를 집행"

규모감 표현:
- "지하부터 지상 7층까지", "하루 평균 11만 명이 방문"
- "약 4만 개, 메가 돈키호테에서는 6만~10만 개 상품"
- "전국에 약 4,700개 점포, 약 17만 개의 디지털 화면"
```

### 수치 전달 패턴

수치를 제시할 때는 단순 나열이 아니라 **"전 → 후"** 또는 **"비교 대상"**과 함께 맥락을 부여한다:
```
- "평균 18시간에서 4.4시간으로" (전후 비교)
- "2023년 24% 대비 약 2배 성장" (연도 비교)
- "2022년 22.7억 달러에서 2026년 약 58억 달러로 2.5배 이상" (성장 추이)
```

## 8. 자사 제품(매쉬) 연결법

### 연결 위치

글의 **마지막 섹션(H2)**에서 자연스럽게 전환한다. 해외 사례의 교훈을 한 문장으로 정리한 뒤 매쉬로 연결하는 것이 표준이다.

### 연결 흐름: 사례 교훈 요약 → 일반화 → 매쉬 소개 → CTA

```
[사례 교훈 요약]
"돈키호테는 매장 운영 과정에서 발생한 페인 포인트를 해결하기 위해 리테일 AI를 도입했습니다."

[일반화 + 매쉬 연결]
"오프라인 매장 방문객 솔루션 매쉬(mAsh)도 이렇게 쓸 수 있습니다."

[매쉬의 가치]
"감에만 의존해 알 수 없던 매장 데이터를 알아내고, 데이터 기반 의사결정을 할 수 있습니다."

[CTA]
"자사 매장의 첫 번째 리테일 AI로 매쉬를 도입해보세요."
```

### 연결 시 주의사항

- 매쉬를 직접 광고하지 않고, 해외 사례의 교훈에서 **자연스럽게 이어지도록** 한다.
- "매쉬도 이렇게 쓸 수 있습니다", "매쉬가 그 첫걸음이 될 수 있습니다" 처럼 사례와 매쉬를 **동일선상에 놓는 표현**을 사용한다.
- 매쉬 소개는 2~4문장 이내로 간결하게 유지한다.
- "매장 방문객 분석", "딥러닝 기반 분석", "매장 데이터" 등의 키워드로 매쉬의 핵심 기능을 언급한다.

### 연결 소제목 예시

```
- "리테일 AI와 경험의 조화, 매장 방문객 분석으로 시작해보세요"
- "매장 첫 번째 리테일 AI는 매쉬와 함께하세요"
- "리테일 AI 성공 사례를 메이아이와 함께 만들어보세요"
- "AI 기반 서비스 혁신이 성과 레버리지의 지름길!"
```

## 9. CTA(행동 유도) 패턴

### CTA 위치와 구성

CTA는 글의 **마지막 단락**에 위치하며, 아래 요소를 순서대로 포함한다:

1. 서비스 소개서 안내 (가장 주된 CTA)
2. 채널톡/이메일 안내 (보조 CTA)

### 실제 CTA 문장 예시

```
- "아래 버튼을 누르면 자세한 설명이 담긴 서비스 소개서를 읽어볼 수 있습니다."
- "매쉬에 대한 더 자세한 소개는 서비스 소개서를 받아보세요."
- "아래 버튼을 눌러 서비스 소개서를 받아보세요."
- "자세한 상담이 필요한 분들은 우측 하단 채널톡 또는 contact@may-i.io로 연락 부탁드립니다."
- "도입 문의는 우측 아래 채널톡 버튼 또는 contact@may-i.io로 상담 부탁드립니다."
- "도입 문의는 contact@may-i.io로 연락해 주세요."
```

### CTA 톤

- 강압적이지 않고, **제안하는 톤**을 유지한다: "~해보세요", "~받아보세요", "~부탁드립니다"
- "지금 바로!", "놓치지 마세요!" 같은 과도한 긴급성 표현은 사용하지 않는다.

## 10. 소제목 스타일

### H2 소제목 (대단원)

- **질문형:** "돈키호테는 어떤 기업일까?", "리테일 AI는 오프라인 매장에 어떻게 쓰이고 있을까?"
- **번호 포함 선언형:** "리테일 AI 첫 번째, AI 가격 책정 시스템", "AI 리테일 테크 두 번째: 클럽카드 데이터와 AI의 결합"
- **핵심 메시지형:** "AI 기반 서비스 혁신이 성과 레버리지의 지름길!", "리테일 AI 도입 우선순위, 고객에게서 길을 찾는다"
- 사례 나열 시 "첫 번째", "두 번째", "세 번째"를 소제목에 직접 명시한다.

### H3 소제목 (세부 항목)

- **구체적 내용 요약:** "테스코의 누적된 물류 공급망 문제", "리테일 테크로 공급망 문제를 해결한 방법"
- **기존 문제 → 해결 구조:** "기존 플래노그램의 문제점" → "리테일 AI로 플래노그램 효율화하는 법"
- **질문형:** "REWE의 Pick and Go, 어떤 원리일까?", "스마트 쇼핑 카트, 어떤 기능이 있을까?"

### 소제목 작성 규칙

- 명사형 종결보다 서술형/질문형 종결을 선호한다.
- 기술 용어와 일상 언어를 함께 사용한다: "AI 플래노그램" + "제품 진열하는 작업"
- 소제목만 읽어도 글의 흐름을 파악할 수 있어야 한다.

## 11. 문장 길이와 단락 구성

### 문장 길이

- 평균 문장 길이는 **30~60자** 수준이다.
- 짧은 문장(20자 이내)과 긴 문장(80자 이상)을 섞어 리듬감을 준다.
- 핵심 메시지는 짧고 강하게, 설명은 길고 부드럽게 쓴다.

### 단락 구성

- 한 단락은 **2~4문장**으로 구성한다.
- 긴 설명이 필요할 때는 **불릿 포인트(- )나 번호 목록(1. 2. 3.)**으로 전환한다.
- 불릿 포인트 안의 항목도 2~3문장으로 충분히 설명한다 (단순 나열이 아님).

### 강조 표현

- **볼드(`**텍스트**`)**를 적극 활용한다. 한 섹션에 1~3개의 볼드 문장이 들어간다.
- 볼드는 주로 다음에 사용한다:
  - 성과 수치가 포함된 문장
  - 핵심 인사이트 또는 결론 문장
  - 자사 제품 연결 문장
- 괄호를 활용한 부연: "(업계 4위)", "(약 363평)", "(mAsh)" 등

## 12. 전체 글 길이

### 분량 기준

- **총 길이:** 약 1,200~2,000자 (공백 제외 기준), 마크다운 기준 약 60~80줄
- **섹션 수:** H2 기준 4~6개
- **사례 수:** 해외 기업 사례 글의 경우 2~4개 사례를 다룸
- **읽는 데 걸리는 시간:** 약 5~8분

### 섹션별 대략적 비중

| 섹션 | 비중 |
|------|------|
| 도입부 | 10~15% |
| 기업/시장 배경 | 15~20% |
| AI 활용 사례 (본문) | 50~60% |
| 자사 연결 + CTA | 10~15% |

## 13. 기타 스타일 규칙

### 외래어/영어 표기

- 기업명은 한글과 영문을 병기한다: "테스코(Tesco)", "던험비(Dunnhumby)", "매쉬(mAsh)"
- 기술 용어는 첫 등장 시 풀어서 설명한다: "플래노그램(제품 진열하는 작업)", "퍼스트 파티 데이터(직접 수집한 고객 데이터)"
- 이후 반복 시에는 한글 또는 약어만 사용한다.

### 마크다운 서식

- H1(`#`): 제목 (1개)
- 제목 아래: 부제목 (볼드 또는 일반 텍스트)
- `---`: 각 대섹션 사이 구분선
- H2(`##`): 대단원 소제목
- H3(`###`): 세부 소제목
- `**볼드**`: 핵심 문장 강조
- `- ` 불릿: 나열/비교 항목
- `1. 2. 3.`: 순서가 있는 프로세스 설명

### 톤 체크리스트

글을 작성한 뒤 아래 항목을 확인한다:

- [ ] 해요체/합니다체가 자연스럽게 혼용되어 있는가?
- [ ] "~고요" 패턴이 적절히 사용되어 가벼운 톤을 유지하는가?
- [ ] 해외 사례의 성과 수치가 볼드 처리되어 있는가?
- [ ] 한국 독자가 공감할 수 있는 맥락(환산, 비교)이 포함되어 있는가?
- [ ] 매쉬 연결이 사례 교훈에서 자연스럽게 이어지는가?
- [ ] CTA가 제안하는 톤으로 작성되어 있는가?
- [ ] 소제목만으로 글의 전체 흐름을 파악할 수 있는가?
- [ ] 불릿 포인트와 번호 목록이 적절히 활용되어 있는가?
