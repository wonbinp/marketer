# 블로그 제품소개 글 스타일 가이드

> 메이아이 블로그의 제품소개(products) 카테고리 글 작성 시 참고하는 스타일 가이드.
> 22편 이상의 기존 글을 분석하여 도출한 패턴이다.

---

## 1. 문체

### 기본 어미
- **"~입니다/~습니다"** 체(합쇼체)를 기본으로 사용한다.
- 중간중간 **"~인데요", "~거예요", "~있답니다", "~거죠", "~같아요"** 등 부드러운 해요체를 섞어 딱딱함을 완화한다.
- 하나의 단락 안에서 합쇼체와 해요체를 자연스럽게 오간다.

### 격식 수준
- 전문성과 친근함의 균형. B2B 콘텐츠이지만 지나치게 딱딱하지 않다.
- 전문 용어(KPI, 전환율, 피플 카운팅, 퍼널 등)를 자연스럽게 쓰되, 필요하면 곧바로 풀어서 설명한다.

### 어미 패턴 예시
```
- "~할 수 있습니다." (기본 서술)
- "~하기 때문인데요." (설명 연결)
- "~이라고 할 수 있습니다." (정의/요약)
- "~할 수 있겠습니다." (제안)
- "~아닐까 생각됩니다." (부드러운 추측)
- "~하는 거예요." / "~하는 거죠." (구어체로 친근하게)
- "~해보세요." (권유)
- "~부탁드립니다." (정중한 요청)
```

## 2. 글 구조 (도입 -> 본문 -> 마무리)

### 전체 흐름
```
[제목]
[부제 또는 요약 한 줄]
---
[인사 + 도입부: 문제 제기 또는 상황 설정] (1~3단락)
---
[본문 섹션 1: 문제/현황 설명] (## 소제목)
[본문 섹션 2: 솔루션 소개 및 연결] (## 소제목)
[본문 섹션 3~4: 구체적 활용법/데이터] (## 소제목)
---
[마무리 섹션: CTA 포함] (## 소제목)
```

### 구조 특징
- 본문 섹션은 보통 **3~5개의 ##(h2) 소제목**으로 구성된다.
- 각 섹션 사이에 `---` 수평선을 넣어 시각적으로 구분한다.
- 도입부에서 문제를 제기하고, 본문에서 해결책을 제시하며, 마무리에서 행동을 유도하는 **문제-해결-행동** 3단 구조를 따른다.

---

## 3. 제목 작성법

### 패턴 유형

**질문형** (가장 빈번):
```
- "매장 방문객도 웹사이트 방문객처럼 분석할 수 있다고요?"
- "매장 방문객 분석을 직원 관찰에만 의존하고 있나요?"
- "마스크를 쓴 매장 방문객도 분석이 가능할까요?"
- "개인정보보호법 때문에 매쉬 도입이 망설여지시나요?"
- "매장 방문객 데이터 분석 솔루션을 사용하면 매출이 오를까요?"
```

**선언/공개형**:
```
- "공개합니다, 메이아이 고객사가 모니터링 하는 매장 관리 KPI 6개!"
- "매쉬를 활용한 인사이트 보고서, 당신도 만들 수 있다!"
- "2분 만에 업무 생산성을 올려주는 매쉬보드 기능 3종"
```

**방법 제시형**:
```
- "피팅룸 데이터 관리로 매출 2배 내는 고객 늘리는 방법은?"
- "고객이 열광하는 뷰티 매장, 방문객 데이터 분석으로 만들 수 있습니다"
- "팝업스토어 기획 효과, 방문객 체류 시간으로 측정할 수 있다!"
```

**비교형**:
```
- "매쉬 vs. 피플카운터 솔루션 4종 비교!"
```

### 제목 작성 규칙
- 독자의 실무 고민이나 니즈를 제목에 직접 담는다.
- "매장", "방문객", "데이터", "분석" 등 핵심 키워드를 포함한다.
- 숫자를 활용하면 구체성이 높아진다 (예: "KPI 6개", "기능 3종", "매출 2배").
- 느낌표(!) 또는 물음표(?)로 끝내 주목도를 높인다.

## 4. 부제(서브 타이틀)

제목 바로 아래에 한 줄짜리 부제를 배치한다. 볼드, 이탤릭, 또는 인용 블록(>) 형식을 사용한다.

```
# 팝업스토어 마케팅 효과, 제대로 측정하고 계시나요?
> 기대와 추측이 아닌 정량적 성과를 집계하는 인공지능 데이터 분석 솔루션 매쉬

# 매장 방문객도 웹사이트 방문객처럼 분석할 수 있다고요?
**오프라인 공간의 Google Analytics, 매쉬만 있다면 가능합니다**

# 피팅룸 데이터 관리로 매출 2배 내는 고객 늘리는 방법은?
**부제: 패션 매장 KPI 개선 방법 3가지를 소개해요**
```

### 부제 역할
- 제목이 질문이면 부제에서 방향성을 제시한다.
- 글의 핵심 메시지를 한 문장으로 압축한다.
- "매쉬"나 "메이아이"를 부제에서 처음 언급하는 경우가 많다.

## 5. 도입부 패턴

### 인사 + 자기소개
대부분의 글은 아래 형식으로 시작한다:

```
안녕하세요, 메이아이의 [직함] [이름]입니다.
안녕하세요, 오프라인 공간을 더 똑똑하게 만드는 인공지능 스타트업 메이아이입니다.
```

- 필자 이름을 밝히는 경우: "메이아이의 Brand Marketer 고운", "메이아이의 Content Editor 최창근", "메이아이의 Business Development Manager 공태웅", "메이아이의 Product Designer 정두리" 등
- 회사명으로 시작하는 경우: "오프라인 공간을 더 똑똑하게 만드는 인공지능 스타트업 메이아이" (정형화된 수식어)

### 도입 전개 방식

**1) 질문 던지기 (가장 흔함)**:
```
"메이아이의 매쉬를 도입한 대기업 고객사는 어떤 KPI를 매일 모니터링할까요?"
"오프라인 매장에 방문객이 얼마나 오는지 측정하려고 계획 중인가요?"
"CCTV 영상 속 방문객을 분석하는 인공지능 매쉬는 태생적으로 개인정보 보호의 개념과 밀접한 관계를 갖습니다."
```

**2) 트렌드/현상 제시**:
```
"요즘에는 외식할 때 뷔페에 가는 게 트렌드입니다."
"시장조사기관 엠브레인 집계에 따르면 전국 만 19~59세 성인 남녀 10명 중 8명이..."
"팝업스토어도 MBTI처럼 유형이 다양하게 나뉘는 시대입니다."
```

**3) 공감 상황 설정**:
```
"성공적인 경영은 자사 데이터를 정확히 측정하는 데서 시작합니다."
"퇴근하는 직장인 중 '오늘 저녁에는 피자나 시켜 먹자'라며 2만 원가량 호쾌하게 지출하실 분은 얼마나 될까요?"
```

**4) 자주 받는 질문 활용**:
```
"매쉬를 소개할 때마다 빠지지 않고 나오는 질문이 있습니다. '매쉬를 사용하면 매장의 매출이 오르나요?'라는 질문입니다."
"방문객 분석 솔루션을 접한 많은 분들은 '매쉬는 매장 방문객이 마스크를 써도 잘 분석하나요?'라고 묻습니다."
```

### 도입부 분량
- 인사 후 1~3단락으로 구성한다.
- 문제 상황을 설정한 뒤 "이번 콘텐츠에서는 ~을 소개합니다" 형태로 글의 방향을 예고하며 마무리한다.

## 6. 독자 호칭

- **"매장 관리자"**, **"마케터님"**, **"담당자님"** 등 직무 기반 호칭을 사용한다.
- "고객사", "고객사 담당자님" 등 B2B 맥락의 호칭도 쓴다.
- **"~하고 계신 분"**, **"~을 고민하는 분"** 처럼 상황 기반으로 지칭하기도 한다.
- 직접적인 2인칭("당신")은 드물게 사용한다. 쓸 경우 "당신도 만들 수 있다!" 처럼 임팩트 있는 맥락에서만 쓴다.

### 호칭 예시
```
- "오프라인 마케팅의 성공 사례를 만들기 위해 혼신의 힘을 다하고 계신 마케터님!"
- "방문객 데이터 분석에 고민이 많은 매장 관리자라면"
- "매장 방문객 데이터에 대한 고민이 있는 F&B 업계 관계자라면"
- "매쉬의 도입을 적극적으로 추진하고 계신 고객사와 담당자님"
```

## 7. 제품(매쉬) 소개 연결법

### 연결 전략
매쉬를 갑자기 소개하지 않는다. **문제 -> 한계 -> 매쉬가 해결** 구조로 자연스럽게 연결한다.

### 3단계 연결 공식

**Step 1: 기존 방식의 한계를 짚는다**
```
"사람의 눈으로 관찰하는 방법은 주관이 많이 섞이는 한계가 있습니다."
"수동 계수 방식은 정확도가 33% 감소합니다."
"포스기를 통해 방문객 데이터를 확인하는 방법은 정확도가 40% 수준입니다."
```

**Step 2: 매쉬를 해결책으로 제시한다**
```
"매쉬를 활용하면 테이블 앞 체류 시간을 측정해 인기 정도를 파악할 수 있습니다."
"인공지능 기반 매장 방문객 분석 솔루션 '매쉬'를 활용하는 관리자는 실무자 보고에만 의존하지 않습니다."
"메이아이는 '오프라인 공간의 Google Analytics'라 불리는 매장 방문객 분석 솔루션 매쉬(mAsh)를 제공합니다."
```

**Step 3: 구체적 기능과 수치를 덧붙인다**
```
"매쉬 피플 카운팅 기능으로 실제 방문객 데이터를 95% 수준의 정확도로 수집할 수 있습니다."
"직원 필터링 기술, 방문객의 체류 시간 집계, 동선 추적 등 매쉬의 핵심 기능과 인사이트를 모두 받아볼 수 있습니다."
```

### 매쉬 표기
- 처음 등장 시: **매쉬(mAsh)** 또는 **'매쉬'** 로 표기한다.
- 이후에는 "매쉬"로만 표기한다.
- 대시보드는 **"매쉬보드"** 라고 부른다.
- 팝업스토어 전용 상품은 **"팝업 플랜(Pop-up Plan)"** 이라고 부른다.

### 매쉬의 정형 수식어
```
- "인공지능 기반 매장 방문객 분석 솔루션"
- "매장 분석 AI 솔루션"
- "오프라인 공간의 Google Analytics"
- "오프라인 공간을 더 똑똑하게 만드는"
```

### 자사 차별점 강조 표현
```
- "경쟁사의 추종을 불허하는, 메이아이라서 제공할 수 있는 독보적인 데이터"
- "오직 매쉬에서만 제공할 수 있는"
- "메이아이만이 제공할 수 있는 독보적 지표"
- "앞서 소개한 솔루션의 단점을 모두 보완하고, 월등한 성능을 자랑합니다"
```

## 8. CTA(행동 유도) 패턴

### 마무리 섹션 구성
마지막 ## 소제목 아래에 1~2단락으로 CTA를 배치한다.

### CTA 소제목 스타일
요약 메시지나 핵심 가치를 담은 선언형 문장:
```
- "메이아이의 고객사는 관성을 버렸습니다"
- "오프라인 방문객도 데이터로, 한눈에!"
- "공들인 행사의 가치, 추측하지 말고 측정하세요"
- "피플 카운팅은 AI 기반 솔루션이 제대로 합니다"
- "손님 한 명 한 명의 가치를 아는 브랜드라면"
- "매장 이탈 방문객을 잡아야 진짜 매장입니다"
```

### CTA 문장 패턴
```
- "서비스 소개서를 받아보세요." / "소개서를 확인해 보세요."
- "채널톡 또는 contact@may-i.io로 문의 부탁드립니다."
- "어떤 문의 사항에도 친절하고 빠르게 응답해 드리겠습니다 :)"
- "궁금한 점을 채널톡으로 전달해 주셔도 좋습니다."
- "빠르고 성실하게 답변드리겠습니다."
- "담당자가 궁금한 점을 다 해소해드리겠습니다."
```

### CTA 구성 공식
```
[핵심 가치 요약 1~2문장]
+ [소개서 받기 권유]
+ [채널톡/이메일 안내]
+ [친절한 응대 약속]
```

### CTA 예시 전문
```
더 자세한 내용이 궁금하다면 아래 버튼을 눌러 서비스 소개서를 받아보세요.
솔루션 비용 및 도입 과정에 대해 질문이 있다면 contact@may-i.io 또는
우측 하단 채널톡 버튼을 통해 연락 부탁드립니다. 담당자가 궁금한 점을 다
해소해드리겠습니다.
```

## 9. 소제목(## h2) 스타일

### 패턴 유형

**질문형**:
```
- "방문객 수만으로 매장 전체 맥락을 설명할 수 있을까?"
- "매장 이탈률은 만능 데이터일까?"
- "피팅룸 운영 핵심 데이터 리스트, 궁금한가요?"
- "뷔페 매장에 방문객 데이터 분석 솔루션이 필요한 이유는?"
```

**선언/요약형**:
```
- "그야말로 가장 똑똑한 파트너"
- "매쉬는 '정확한 데이터'를 남겨드립니다"
- "'이것' 덕분에 가능합니다"
- "이미 온라인에서는 당연한 이야기"
- "이제 오프라인에서도 가능한 이야기"
```

**행동 유도형**:
```
- "매장 방문객 분석, 소개서를 받으면 바로 시작됩니다"
- "더 다양한 데이터, 매쉬로 발견하고 개선해보세요"
```

**번호 + 설명형** (리스트형 글에서):
```
- "1. 동행 여부 판단"
- "2. 주요 동선 시각화"
- "첫 번째 유형: 스탬프 투어형 팝업스토어"
```

### 소제목 작성 규칙
- 한 글에 3~6개의 h2 소제목을 사용한다.
- 본문 구조에 따라 질문형과 선언형을 혼합한다.
- 도입 섹션은 질문형, 해결책 섹션은 선언형, 마무리는 행동 유도형으로 배치하는 흐름이 자연스럽다.
- h3(###)은 h2 아래 세부 항목을 나열할 때 사용한다.

---

## 10. 불릿포인트/리스트 활용

### 활용 장면
- 기존 방식의 **단점 나열** 시
- 매쉬의 **기능/장점 나열** 시
- **구체적 데이터 항목** 정리 시
- **활용 방법 단계** 설명 시

### 불릿 스타일
- `-` 기호로 시작하는 비순서 목록을 주로 사용한다.
- 순서가 중요한 경우 `1. 2. 3.` 번호 목록을 사용한다.
- 각 항목에 **볼드로 키워드를 강조**한 뒤 콜론(:)으로 설명을 잇는 패턴이 빈번하다.

### 불릿 패턴 예시
```
- **높은 정확도:** 95% 이상의 정확도로 방문객 수를 집계합니다.
- **방문객 프로파일링:** 방문객의 성별과 연령대 비중을 분석해 제공합니다.
- **도입 공사 필요 없음:** 매장 내 설치된 CCTV를 그대로 활용합니다.
```

### 볼드 강조 규칙
- 핵심 메시지나 결론 문장에 **볼드**를 적용한다.
- 한 단락에 1~2개의 볼드 문장을 넣는다.
- 볼드는 문장 전체 또는 핵심 구절에 적용한다.

## 11. 문장 길이와 단락 구성

### 문장 길이
- 평균 문장 길이: **40~80자** (한 줄~두 줄)
- 짧은 문장과 긴 문장을 번갈아 사용해 리듬감을 준다.
- 지나치게 긴 문장(100자 이상)은 쉼표나 마침표로 분절한다.

### 단락 구성
- 한 단락은 **2~4문장**으로 구성한다.
- 한 소제목 아래 2~4단락이 오는 것이 일반적이다.
- 단락 사이에 빈 줄을 넣어 가독성을 확보한다.

### 접속/전환 표현
단락 간 연결에 자주 쓰는 표현:
```
- "그런 의미에서"
- "이해를 돕기 위해"
- "구체적으로 예를 들면"
- "따라서"
- "하지만"
- "그래서인지"
- "이와 달리"
- "또한"
- "덧붙이자면"
- "물론"
- "이처럼"
```

## 12. 전체 글 길이

- 일반 글: **1,000~2,000자** (원고지 5~10매 분량)
- 심화 글(KPI 시리즈, 솔루션 비교 등): **2,000~3,500자**
- h2 소제목 기준 **3~6개 섹션**으로 구성

## 13. 자주 쓰는 표현 및 관용구

### 반복 등장 표현

**문제 제기 시:**
```
- "~에는 큰 맹점이 하나 있습니다"
- "~하기 어려운/어려웠습니다"
- "~에 의존할 수밖에 없었기 때문입니다"
- "~라는 한계가 있습니다/존재했습니다"
- "정성적 평가만을 받아왔죠"
- "정확하지 않은 데이터는 잘못된 의사결정의 원인이 됩니다"
```

**해결책 제시 시:**
```
- "매쉬를 활용하면"
- "매쉬를 도입하면"
- "메이아이는 ~를 제공합니다"
- "데이터를 기반으로"
- "정량적으로 측정할 수 있는"
- "95% 수준의 정확도로"
```

**가치 강조 시:**
```
- "결과적으로 매출 향상이라는 목표에 더 빠르게 도달할 수 있습니다"
- "합리적 의사결정에 목마른"
- "온라인에서는 이미 당연한"
- "이제 오프라인에서도 가능한"
- "감에 의존하던 ~은 과거가 되었습니다"
- "데이터라는 저울"
```

**친근함/구어체:**
```
- "~하는 거예요" / "~하는 거죠"
- "~하기 마련입니다"
- "~할 수도 없거니와"
- "~하기에 제격인"
- "속상한 일은 없을 겁니다"
- "알고 계시나요?"
```

**마무리 격려:**
```
- "좋은 인연은 작은 관심으로부터 시작됩니다"
- "매쉬는 간단하고 합리적이며 믿을 수 있습니다"
- "매장 성과를 높이는 일에 날개를 달아보세요"
- "어떤 궁금증도 성실하게 해결해 드릴 준비가 되어 있습니다"
```

### 비유/수사법
```
- "웹사이트에 구글 애널리틱스를 연결하는 것이 당연해진 것처럼" (온라인 비유)
- "데이터라는 저울" (은유)
- "아끼는 식물에 물을 주는 마음으로" (감성 비유)
- "가장 똑똑한 파트너" (의인화)
```

## 14. 특수 서식 요소

### 인용 블록
부제, 법률 조항 인용, 핵심 인사이트 강조에 사용한다:
```
> 기대와 추측이 아닌 정량적 성과를 집계하는 인공지능 데이터 분석 솔루션 매쉬
```

### 꺾쇠 인용(「」)
핵심 인사이트나 요약 문장에 사용한다:
```
「체험·이벤트존에 관심을 갖고 꼼꼼히 살펴본 방문객의 만족도가 반영되지 못하니 기획의 성패 여부도 제대로 측정하기 어려운 셈입니다.」
```

### 표(Table)
솔루션 비교, 측정 방법 비교 시 활용한다:
```
| 방법 | 특징 |
|---|---|
| 수동 클릭 계수기 | 스태프가 직접 카운팅 |
| 적외선 계수기 | 저렴하나 한계 존재 |
| CCTV 기반 AI 솔루션 | 성별·연령대 측정 가능 |
```

### 시리즈 안내
시리즈 글의 경우 글 말미에 시리즈 목록을 배치한다:
```
> **매장 KPI 기초 시리즈**
> - 1탄: 방문객 수의 거의 모든 것
> - 2탄: 이탈률의 거의 모든 것
```

## 15. 서브 카테고리별 스타일 차이

### KPI 시리즈 (product-kpi-*)
- **교육/가이드 톤**: 개념 설명 -> 한계점 -> 측정 방법 -> 활용법 순서
- 부제에 "~의 거의 모든 것" 패턴 사용
- 볼드 + 불릿 조합으로 정보를 구조화
- 인사 없이 바로 본문으로 들어가는 경우도 있음
- CTA가 짧거나 없는 경우도 있음 (교육 콘텐츠 특성)

### 데이터 기반 운영 (product-datadriven-*)
- **설득/제안 톤**: 현재 문제 -> 매쉬의 해결 -> 구체적 효과
- 업종별 사례(뷔페, 뷰티 등)를 도입부에 배치해 관심 유도
- "~하면 ~할 수 있습니다" 패턴의 제안형 문장이 많음
- 필자(Content Editor, Brand Marketer) 자기소개가 거의 항상 포함

### 팝업스토어 (product-plan-*)
- **실무/현장 톤**: 현장 담당자의 고충 공감 -> 매쉬 팝업 플랜 소개
- 숫자와 구체적 프로세스를 많이 활용 (3주, 95% 등)
- "팝업 플랜", "임시 보안 카메라 키트" 등 제품명 강조
- 팝업 유형별 세분화된 KPI 추천이 특징

### 대시보드 (product-mashboard-*)
- **튜토리얼/가이드 톤**: 기능 소개 -> 사용법 안내
- 단계별(Step 1, 2, 3) 설명이 많음
- "~탭을 클릭합니다", "~버튼을 누릅니다" 등 조작 안내 문체
- 프로덕트 팀의 고민과 노력을 서사적으로 풀어냄

### 개인정보 (product-privacy-*)
- **안심/신뢰 톤**: 법률 조항 인용 -> 매쉬의 준수 방식 -> 도입 절차
- 법률 용어와 조항 번호를 정확히 명시
- "~하셔도 괜찮습니다", "~염려는 접어두셔도 괜찮습니다" 등 안심 표현
- 실제 고객사 사례(신세계백화점, CGV)를 근거로 제시

### 솔루션 비교 (product-solutionvs-*)
- **객관적 비교 톤**: 각 솔루션의 장단점을 동일 구조로 나열
- h2 + h3 조합으로 "장점" / "단점"을 분리
- 마지막에 매쉬를 배치해 자연스럽게 우위를 보여줌
- 구체적 수치(68% 비용 절감, 95% 정확도, 정확도 33% 감소 등)를 적극 활용

### 패션 (product-fashion-*)
- 피팅룸, 매대 등 패션 업종 특화 용어 활용
- 외부 연구/통계 데이터를 도입부에 인용 (구매 확률 67% 증가 등)
- 벤치마크 수치(피팅룸 방문 비율 25%, 체류 시간 4분 29초 등) 제공

## 16. 작성 시 주의사항

### 피해야 할 것
- 지나치게 딱딱한 학술 문체
- 과도한 영어 사용 (필수 전문용어 외)
- 매쉬를 도입부터 전면에 내세우는 직접적 광고 톤
- 근거 없는 과장 표현
- 독자를 낮추어 보는 듯한 어조

### 지켜야 할 것
- 독자의 실무 고민에서 출발한다
- 문제를 충분히 공감한 뒤 해결책을 제시한다
- 수치와 사례로 구체성을 확보한다
- "~해보세요", "~확인해보세요" 등 부드러운 권유로 마무리한다
- 볼드 강조로 핵심을 짚어주되 남용하지 않는다
- 온라인과 오프라인을 대비시키는 프레이밍을 적극 활용한다
