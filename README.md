# Marketer — 메이아이 콘텐츠 생성 봇

메이아이(mAy-I)의 **보도자료와 블로그 글을 기존 스타일과 일관되게 작성**하는 콘텐츠 생성 도구입니다. Claude Code 위에서 동작하며, 유형별 스타일 가이드와 과거 원문을 참고해 톤·구조·문체를 맞춰 콘텐츠를 생성합니다.

## 사용법

Claude Code에서 콘텐츠 작성을 요청하면(슬래시 커맨드 없이도) 자동으로 워크플로우를 따릅니다.

- 예: "보도자료 써줘", "블로그 하나 작성해줘", "투자 유치 콘텐츠 만들어줘", "고객 사례 글 쓸 건데"

내부적으로 `.claude/commands/write-content.md`의 워크플로우를 읽고 동일하게 수행합니다.

## 디렉토리 구조

```
marketer/
├── data/
│   ├── articles/          ← 보도 기사 원문 (유형별: investment, awards,
│   │                         performance, customers, products, government, interviews)
│   └── blog/              ← 블로그 원문 (customers, products, insights)
├── styles/                ← 유형별 스타일 가이드
│   ├── press-release-*.md ← 보도자료 스타일 (7종)
│   └── blog-*.md          ← 블로그 스타일 (3종)
├── .claude/commands/      ← 슬래시 커맨드
└── CLAUDE.md              ← 프로젝트 규칙·작성 원칙
```

## 작성 원칙

1. **스타일 가이드를 먼저 읽는다.** 해당 유형의 `styles/` 파일에서 문체·구조·표현을 따른다.
2. **원문을 참고한다.** `data/`의 원문 2~3편을 few-shot 예시로 읽어 톤과 수준을 맞춘다.
3. **유형이 섞이면** 관련 스타일 가이드를 모두 참고하되 주된 유형을 기본으로 한다.
4. **문체 구분** — 보도자료는 `~했다` 서술체, 블로그는 `~입니다/~해요` 혼용체.
5. **과장하지 않는다.** 제공된 팩트만으로 작성하고 없는 정보를 지어내지 않는다.
