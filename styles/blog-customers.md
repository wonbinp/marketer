# 블로그 고객사례 스타일 가이드

> 이 문서는 메이아이 블로그의 고객사례(Customer Story) 10편을 분석하여 도출한 스타일 가이드입니다. 글 생성 시 시스템 프롬프트로 활용할 수 있습니다.

---

## 1. 문체

### 어미 패턴: ~입니다/~습니다 기반, 도입부만 ~해요체 혼용

고객사례 글은 두 가지 문체가 교차합니다.

- **필자 서술 파트** (도입부, 분석 해설, 마무리): `~입니다/~습니다` 격식체를 기본으로 하되, 도입부에서 독자와 거리를 좁힐 때 `~해요/~인데요/~있어요` 해요체를 섞어 씁니다.
- **인터뷰 파트** (고객사 담당자 발언): `~했어요/~이에요/~거든요/~있습니다` 등 자연스러운 구어체를 그대로 살립니다.

#### 필자 서술 예시

```
"K팝 팬들이 꼭 모으는 아이템, 바로 포토카드인데요."
"이번 콘텐츠에서는 인플루디오가 [...] 어떻게 데이터 기반으로 혁신했는지 전합니다."
"정확하지 않은 데이터는 의사결정에 활용할 수 없습니다."
```

#### 인터뷰 발언 예시

```
"초기에는 모든 고객을 트래킹하는 게 불가능했어요."
"지금은 유동인구가 늘어서 그렇다거나, 매장 내 전환율이 올라서 그렇다거나 하는 식의 매출 변동 원인을 대시보드에서 매일 확인하고 있습니다."
```

### 격식 수준

- 전반적으로 **B2B 전문 매체 수준의 격식**을 유지합니다. 과도한 경어나 학술체는 피하고, 읽기 편한 설명문 톤을 유지합니다.
- "~하셨는데요", "~부탁드립니다" 등 인터뷰 질문에서만 높임을 강화합니다.
- 줄임말, 이모지, 감탄사 등은 사용하지 않습니다.

### 친근함 정도

- 도입부에서 "안녕하세요, 메이아이의 [직함] [이름]입니다"로 시작하여 **필자의 존재감을 드러내며 친근함을 확보**합니다.
- 본문에서는 친근함보다 **신뢰감과 전문성**을 우선합니다.

## 2. 글 구조: 도입 - 본문(인터뷰/분석) - 마무리(CTA)

### 전체 흐름 패턴

```
[제목]
[부제]
---
[도입부: 필자 인사 + 고객사 소개 + 글의 방향 제시]
---
[본문 섹션 1: 도입 전 상황/문제 인식]
[본문 섹션 2: 솔루션 도입 후 변화/활용 사례]
[본문 섹션 3: 성과 및 추가 인사이트]
---
[마무리: 성과 요약 + CTA]
```

### 구조 상세

| 단계 | 역할 | 분량 비중 |
|------|------|-----------|
| 제목 + 부제 | 핵심 성과 또는 질문으로 관심 유도 | - |
| 도입부 | 필자 인사, 고객사 맥락 설명, 글의 방향 제시 | 약 10~15% |
| 본문 섹션 1 | 도입 배경: 기존 문제/과제/니즈 설명 | 약 25~30% |
| 본문 섹션 2 | 활용 사례: 솔루션 도입 후 구체적 변화 | 약 35~40% |
| 본문 섹션 3 | 확장/추가 인사이트 | 약 10~15% |
| 마무리(CTA) | 핵심 요약 + 행동 유도 | 약 5~10% |

## 3. 제목 작성법

### 패턴 1: 질문형 (가장 빈번, 10편 중 7편)

고객사명을 주어로 놓고, 성과 키워드를 포함한 의문문 형태입니다.

```
"인플루디오가 퍼널 분석으로 매장 운영 성과를 개선한 비결은?"
"예스24가 목표보다 14% 초과한 박람회 부스 성과를 측정한 방법은?"
"스파오는 정확도 98%의 매장 방문객 분석 솔루션을 어떻게 활용했을까?"
"아모레퍼시픽은 팝업스토어 운영 성과를 어떻게 분석하고 있을까?"
"링크랭크가 방문 전환율을 획기적으로 올릴 수 있었던 까닭은?"
```

### 패턴 2: 성과 강조형 서술문 (10편 중 3편)

구체적 수치나 키워드를 포함한 선언형입니다.

```
"CGV가 영화관 광고 시청률을 16% 개선한 비결"
"메르세데스 벤츠 코리아와 함께한 100일간의 프로젝트"
"놀로스퀘어가 시행 착오는 줄이고 방문객은 늘린 비법"
```

### 제목 작성 규칙

- **고객사명을 반드시 포함**합니다.
- **구체적 수치**(14%, 98%, 16%, 5배, 100일 등)를 가능하면 포함합니다.
- 30~40자 내외가 일반적입니다.
- 제목 아래에 **부제**를 1줄 배치하여 맥락을 보충합니다.

### 부제 예시

```
"포카스팟을 디지털 트랜스포메이션으로 이끈 핵심 데이터 '전환율'"
"RFID 방식의 한계, 매쉬와 빅데이터로 뛰어넘다"
"매대 위치를 전략적으로 바꿔 매출 3배로 올리기"
"2025 서울국제도서전 방문객 분석 사례 A to Z"
```

## 4. 도입부 패턴

### 패턴 A: 필자 인사 + 고객사 맥락 소개 (가장 일반적, 7편)

```
안녕하세요, 메이아이의 Content Editor 최창근입니다.

[고객사/산업에 대한 흥미로운 배경 설명 2~3문장]

[고객사 소개 1~2문장]

이번 콘텐츠에서는 [고객사]가 [핵심 주제]를 어떻게 [동사]했는지 전합니다.
```

#### 실제 예시 (인플루디오)

```
안녕하세요, 메이아이의 Content Editor 최창근입니다.

K팝 팬들이 꼭 모으는 아이템, 바로 포토카드인데요. [배경 설명...]

이러한 지점을 공략해 K팝 팬들이 원하는 포토카드를 모을 수 있도록 다리를 놓아주는 스타트업이 있습니다. [...]

이번 콘텐츠에서는 인플루디오가 [...] 어떻게 데이터 기반으로 혁신했는지 전합니다.
```

### 패턴 B: 바로 고객사/프로젝트 상황 진입 (3편)

인사 없이 고객사의 상황이나 업계 맥락부터 바로 시작합니다.

```
"예스24 브랜드마케팅팀에서 BX 기획을 담당하는 최지원 씨는 오프라인·온라인 행사를 주로 기획한다."
"CGV의 핵심 사업 중 하나는 광고업으로, 극장 인프라를 활용해 관람객에게 광고를 송출하고 광고주로부터 수수료를 받는 구조입니다."
```

### 도입부 핵심 요소

1. **독자의 관심을 끄는 배경 정보** (산업 트렌드, 고객사의 독특한 사업 모델 등)
2. **고객사 소개** (회사명, 주요 서비스, 시장 내 위치)
3. **글의 방향 제시** ("이번 글에서는 ~를 소개합니다" 형태)

## 5. 자주 쓰는 표현/관용구

### 문제 제기 표현

```
"~에는 명확한 한계가 있습니다."
"정확하지 않은 데이터는 의사결정에 활용할 수 없습니다."
"~하는 것은, 인력을 직접 투입하지 않는 한 사실상 불가능했습니다."
"들이는 품에 비해 정확도가 매우 낮았습니다."
"기존에는 감으로만 느꼈던 ~를 이제는 수치로 확인합니다."
```

### 성과/변화 표현

```
"데이터 기반 의사결정"
"데이터 기반으로 매장을 운영하면 성과가 따라옵니다."
"기존 대비 방문객을 N배 늘리는 기염을 토했습니다."
"막연하게 생각하던 '경향성'을 실제 데이터로 확인했습니다."
"데이터 중심의 매장 운영 방식이 자리잡은 게 가장 달라진 점이에요."
"머리로 알던 것을 실제 데이터로 증명한 셈이다."
```

### 전환/연결 표현

```
"~한 이야기를 소개합니다."
"이번 콘텐츠에서는 ~를 전합니다."
"이번 글에서는 ~를 통해 ~를 소개합니다."
"~가 어떻게 ~할 수 있었을까요?"
```

### 솔루션 설명 표현

```
"매쉬를 통해", "매쉬를 활용하여", "매쉬보드에서는"
"대시보드에서 매일 확인하고 있습니다."
"오프라인 데이터를 ~에 활용하고 있습니다."
```

## 6. 인용문 활용

### 형식 1: 인터뷰 Q&A (인플루디오, 현대차 등)

인터뷰어(메이아이)의 질문과 고객사 담당자의 답변을 Q&A 형식으로 교차 배치합니다.

```
**[질문 내용]**

**김:** [답변 내용]

**박:** [답변 내용]
```

- 질문은 볼드 처리합니다.
- 답변자는 이름(성) 약칭 + 콜론으로 표기합니다.
- 답변 중 핵심 문장은 **볼드** 처리합니다.
- 글 초반에 자기소개 질문으로 담당자를 자연스럽게 소개합니다.

### 형식 2: 블록 인용(blockquote) (놀로스퀘어, 메가박스, 현대차 등)

핵심 발언을 `>` 블록 인용으로 별도 강조합니다.

```
> "매쉬를 쓰지 않았더라면 예전처럼 출입구를 하나로 통제하고 수동 계수기를 활용했을 거예요."
> "방문객의 지갑만 잡기보다는 시간과 마음을 잡고 싶습니다."
> "우리는 오늘보다 나은 내일, 미래를 만들기 위해 정확한 데이터가 필요합니다."
```

### 형식 3: 제3자 서술 (스파오, 예스24, 아모레퍼시픽 등)

담당자의 발언을 필자가 간접 화법으로 재구성하여 전달합니다.

```
"98% 이상의 정확도를 원했습니다."
"매쉬를 선택한 또 하나의 이유는 메이아이가 자사 기술에 대해 객관적이고 솔직하게 평가한다는 점이었습니다."
```

### 인용문 핵심 규칙

- 인용 내 **핵심 인사이트 문장은 반드시 볼드 처리**합니다.
- 한 섹션당 1~2개의 볼드 강조 인용이 들어갑니다.
- 인용문은 "문제 인식 → 솔루션 효과 → 미래 비전" 흐름에 맞춰 배치합니다.

## 7. 데이터/성과 표기

### 수치 표기 규칙

- 퍼센트: **볼드 + 숫자%** 형태로 강조합니다.
  - `목표 대비 **14% 초과 달성**`
  - `구매 전환율 → 평상시 대비 **약 15% 감소**`
  - `정확도 **98%**`

- 배수: **N배** 형태로 표기합니다.
  - `방문객을 **5배** 늘리는`
  - `방문자 대비 구매건수 **2.3배**`

- 절대 수치: 숫자 + 단위로 표기합니다.
  - `합산 **12,000명**이 넘는 방문객`
  - `전체 방문객은 **약 4만 명**`
  - `목표치는 **3만 4,000명**`

### 성과 강조 방식

1. **Before/After 대비**: 도입 전후를 명확히 비교합니다.
   ```
   "매쉬를 도입하기 전에는 [...] 정도로만 파악했어요.
   지금은 [...] 매출 변동 원인을 대시보드에서 매일 확인하고 있습니다."
   ```

2. **불릿 포인트 나열**: 여러 성과 지표를 한눈에 보여줍니다.
   ```
   - 방문객 수: 목표 초과 달성
   - 시간대별 방문객: 특정 시간 집중 없이 고르게 분포
   - 크레마존 체류 시간: 목적한 체험 유도 성공
   - 크레마 단말기 현장 판매량도 목표치 초과
   ```

3. **문맥 내 볼드**: 본문 흐름 속에서 핵심 수치만 볼드 처리합니다.
   ```
   "영화 시작 10분 전 착석해 광고를 시청하는 관람객 비율을 기존 대비 **16% 상승**시키는 성과를 냈습니다."
   ```

## 8. CTA(행동 유도) 패턴

마지막 섹션에서 짧게 성과를 요약한 뒤, 행동 유도로 마무리합니다.

### 패턴 A: 서비스 소개서/상담 안내 (가장 일반적)

```
매장 방문객의 전환율 측정에는 매쉬가 가장 적합합니다. 더 자세한 기능이 궁금한 분들은 서비스 소개서를 받아보세요. 비즈니스 맞춤 상담 및 견적 문의는 contact@may-i.io 또는 채널톡 버튼을 통해 부탁드립니다.
```

### 패턴 B: 데모 신청 유도

```
매쉬보드의 '행사 효과 분석' 템플릿에는 메이아이 Data Analyst들의 분석 노하우가 집약되어 있으며, 행사 성과 측정에 필요한 핵심 지표들을 한눈에 확인할 수 있습니다. 데모 신청은 누구나 무료로 가능합니다.
```

### 패턴 C: 자연스러운 권유

```
데이터를 어떻게 수집하고 분석해야 할지 막막하다면 매쉬와 함께하시기를 추천합니다.
```

### 패턴 D: 솔루션 가치 선언으로 마무리

```
매쉬는 성과가 나기 어려운 프로젝트에 리소스가 낭비되는 것을 막고, 성공 가능성이 높은 프로젝트를 빠르게 발견할 수 있도록 돕습니다.
```

### CTA 핵심 규칙

- CTA는 **마지막 섹션**에 배치하되, 분량은 2~4문장으로 짧게 유지합니다.
- 직접적인 "지금 신청하세요!" 같은 압박형 CTA는 절대 사용하지 않습니다.
- "~가 궁금한 분들은", "~하다면 ~를 추천합니다" 등 **부드러운 제안형**을 씁니다.
- CTA 직전에 **고객사 성과를 요약**하여 솔루션 가치를 자연스럽게 환기합니다.

## 9. 소제목 스타일

### 패턴 1: 인용문형 소제목 (인터뷰 기반 글)

고객사 담당자의 핵심 발언을 따옴표로 감싸 소제목에 씁니다.

```
## "K팝 팬이 실제로 방문하는지 검증하고 싶었어요"
## "전환율 데이터 기반으로 매장 운영 전략을 고도화하고 있어요"
```

## 12. 톤 앤 매너

### 핵심 톤: "신뢰감 있는 전문 해설자"

메이아이 블로그 고객사례의 톤은 다음 네 가지 축으로 정의됩니다.

| 축 | 설명 |
|-----|------|
| **전문적** | 오프라인 데이터 분석, 전환율, 히트맵, 퍼널 등 업계 용어를 자연스럽게 사용합니다. 단, 독자가 모를 수 있는 용어는 첫 등장 시 간략히 설명합니다. |
| **객관적** | 과장 표현을 피하고, 구체적 수치와 Before/After 비교로 성과를 제시합니다. "획기적", "놀라운" 같은 형용사보다 숫자로 말합니다. |
| **실용적** | 고객사가 "왜 도입했는지 → 어떻게 활용했는지 → 어떤 성과가 나왔는지"의 실무적 맥락을 전달합니다. |
| **자연스러운** | 딱딱한 보도자료가 아니라 이야기를 들려주는 방식으로 서술합니다. 인터뷰 형식을 통해 현장감을 살립니다. |

### 피해야 할 톤

- 과도한 자사 제품 찬양 ("최고의", "완벽한", "혁신적인")
- 압박적 판매 어조 ("지금 바로 도입하세요!")
- 학술적/논문체 ("~에 관한 연구 결과를 토대로")
- 지나친 구어체/유행어 ("대박", "미쳤다", "갓")

### 자사 제품 언급 방식

- 제품명 '매쉬(mAsh)' 또는 '매쉬보드'를 자연스럽게 문맥 속에서 언급합니다.
- 고객사가 직접 효과를 설명하는 형태로 제품 가치를 전달합니다 (자사 주장이 아닌 고객 증언).
- "매쉬를 통해 ~할 수 있게 되었습니다"가 가장 자주 쓰이는 패턴입니다.
