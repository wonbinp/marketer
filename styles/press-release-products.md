# 제품 출시/업데이트 보도자료 스타일 가이드

> 이 문서는 메이아이의 신규 제품/기능 출시(매쉬 인사이트 등)를 다루는 보도자료의 작성 스타일을 정의한다. 7편의 실제 보도 기사를 분석하여 도출했다.

---

## 1. 문체

### 어미 패턴
- **발표/사실 전달:** `~했다고 밝혔다`, `~추가했다고 발표했다`
- **기능 설명:** `~할 수 있다`, `~를 제공한다`, `~를 지원한다`
- **계획:** `~할 계획이다`, `~할 방침이다`, `~도입 예정이다`
- **인용 도입:** `~라며`, `~라고 밝혔다`, `~라고 말했다`, `~라고 전했다`

### 격식 수준
- 해라체 격식 평서문 (고객사 보도자료와 동일)
- 기술 설명 부분에서 보다 상세하고 설명적인 톤
- 감성적 표현 없이 기능 중심으로 서술

### 예시
```
메이아이가 자사 방문객 데이터 분석 솔루션 '매쉬'에 대화형 AI 에이전트 기능인 '매쉬 인사이트'를 추가했다고 밝혔다.
```

```
사용자는 웹 대시보드 '매쉬보드'에서 원하는 지표를 골라 질문하면, 대형언어모델(LLM)이 생성한 답변을 받을 수 있다.
```

## 2. 글 구조

### 전형적인 구성 순서

```
1. 리드 (첫 문단) ──── 출시 사실 + 제품명 + 핵심 한 줄 설명
2. (선택) 핵심 요약 ── 3줄 불릿 요약 (일부 매체)
3. 기존 제품 설명 ─── '매쉬란 무엇인가' 배경 설명
4. 신규 기능 상세 ─── 기술 구성, 사용 방법, 차별점
5. 보안/인프라 ────── 엔터프라이즈 보안 관련 설명
6. 향후 계획 ─────── 다음 로드맵 (자동 보고서 등)
7. 담당자 코멘트 ──── 제품팀 리드 인용문으로 마무리
```

### 고객사 보도자료와의 구조적 차이
- **도입 경과(타임라인)가 없다:** 대신 기존 제품 → 신기능의 발전 맥락을 설명
- **기능 설명이 더 상세하다:** 사용 방법(UI 흐름), 기술 스택, 보안 체계까지 다룸
- **향후 계획이 더 구체적이다:** 다음 기능 로드맵을 명시

## 3. 리드(첫 문단) 작성법

### 필수 포함 요소 (등장 순서)
1. **메이아이 소개**: `영상처리 AI 스타트업 메이아이(대표 김찬규)` 또는 `메이아이`
2. **기존 제품명**: `방문객 데이터 분석 솔루션 '매쉬(mAsh)'`
3. **신규 기능명**: `대화형 AI 에이전트 '매쉬 인사이트(mAsh Insights)'`
4. **발표 동사**: `~추가했다고 밝혔다`, `~출시했다고 발표했다`

### 리드 패턴 예시

**패턴 A: 기능 추가 강조형 (가장 빈도 높음)**
```
영상처리 인공지능 스타트업 (주)메이아이(대표 김찬규)가 자사 방문객 데이터 분석 솔루션 '매쉬(mAsh)'에 대화형 AI 에이전트 '매쉬 인사이트(mAsh Insights)' 기능을 추가했다고 3일 밝혔다.
```

**패턴 B: 간결형**
```
메이아이가 자사 방문객 데이터 분석 솔루션 '매쉬'에 대화형 AI 에이전트 기능인 '매쉬 인사이트'를 추가했다고 밝혔다.
```

**패턴 C: 출시 + 발표일 명시형**
```
영상처리 AI 스타트업 메이아이(대표 김찬규)가 방문객 데이터 분석 솔루션 '매쉬(mAsh)'에 대화형 AI 에이전트 기능인 '매쉬 인사이트'를 새롭게 추가했다고 발표했다.
```

### 고객사 보도자료 리드와의 차이
- 계약 상대방(고객사)이 없으므로 자사 제품 중심으로 서술
- `~계약을 체결했다` 대신 `~기능을 추가했다`, `~출시했다` 사용
- 매장 수 등 규모 수치 대신 기능명과 제품명이 핵심 정보

## 4. 자주 쓰는 표현/관용구

### 출시/업데이트 관련
- `~기능을 추가했다고 밝혔다`
- `~를 새롭게 추가했다고 발표했다`
- `~를 출시했다`

### 기존 제품 설명 시
- `자체 개발 딥러닝 기술을 바탕으로` / `자체 개발 딥러닝 기술 기반으로`
- `매장 CCTV 영상을 정밀 분석하고`
- `방문객 동선을 끊김 없이 추적해`
- `방문·구매 전환율 등 핵심성과지표(KPI)를 측정·모니터링하며`
- `KPI 달성과 개선을 위한 방향성을 제안하는 솔루션이다`

### 신규 기능 설명 시
- `거대 언어 모델(LLM)을 결합한 대화형 AI 솔루션`
- `데이터 접근성을 높이고 전략적 의사결정을 지원`
- `현장에서 즉시 활용 가능한 맞춤형 실행 방안을 제시`
- `정보 오류를 최소화`
- `민감한 정보를 선별·보호한 뒤`
- `자체 개발 추론 알고리즘으로 신뢰도 높은 답변을 전달`

### 보안/인프라 관련
- `엔터프라이즈급 보안 체계`
- `강력한 암호화 정책`
- `AWS의 완전 관리형 서비스 '아마존 베드록(Amazon Bedrock)'을 활용`

### 활용 가치 관련
- `마케팅·운영 등 실무에 바로 적용 가능한 전략을 제시`
- `외부 변수와의 인과관계 분석`
- `경영지표 예측 등 심층 분석`
- `매장 운영 최적화와 성과 증대를 지원`
- `업무 효율성을 강화`
- `데이터 분석 및 리포트 작성의 부담을 줄이고`

### 향후 계획 관련
- `추론 알고리즘을 지속적으로 개선하고`
- `자동 보고서 생성 기능을 도입해`
- `고객사와의 협력을 확대하며`
- `AI 에이전트로 발전시켜 나갈 계획`

## 5. 인용문 활용

### 배치 위치
- **보도자료 마지막 단락** (100% 일관적)

### 인용 주체
- **제품 담당자** (이상우 제품팀 리드): 모든 보도자료에서 동일 인물 인용
- 고객사 보도자료와 달리 대표가 아닌 **제품팀 실무 리더**가 코멘트

### 인용문 도입 패턴
```
이상우 제품팀 리드는 "매쉬 인사이트는 메이아이가 오랜 기간 축적한 데이터 분석 노하우에 최신 AI 기술을 결합한 결과물"이라며, 현장에서 즉시 활용 가능한 맞춤형 실행 방안을 제시하는 AI 에이전트로 발전시켜 나갈 계획이라고 밝혔다.
```
```
이상우 제품팀 리드는 "기업 실무자들의 업무 환경과 전문지식을 깊이 반영해 현장에서 즉시 활용 가능한 맞춤형 실행 방안을 제시하는 AI 에이전트로 발전시켜 나갈 계획"이라고 말했다.
```

### 인용문 내용 구성
- **제품의 본질 규정**: `오랜 기간 축적한 데이터 분석 노하우 + 최신 AI 기술의 결합`
- **향후 발전 방향**: `맞춤형 AI 에이전트로 발전시켜 나갈 계획`
- 직접 인용(`"..."`) + `~라며` + 간접 서술 혼합 패턴이 표준

### 고객사 보도자료 인용문과의 차이
| 항목 | 고객사 보도자료 | 제품 보도자료 |
|------|-----------|----------|
| 인용 주체 | 대표(CEO/CPO) | 제품팀 리드 |
| 인용 내용 | 협업 의의 + 포부 | 제품 가치 + 발전 방향 |
| 톤 | 사업적/전략적 | 기술적/실무적 |

## 7. 회사 소개 패턴

### 메이아이 첫 등장 시 수식어 (택 1)
- `영상처리 인공지능 스타트업 (주)메이아이(대표 김찬규)`
- `영상처리 AI 스타트업 메이아이(대표 김찬규)`
- `AI 영상 전문기업 메이아이(대표 김찬규)`
- `메이아이` (간결형, 이미 알려진 매체에서)

### 기존 제품(매쉬) 설명 블록
제품 보도자료에서는 리드 직후에 매쉬 자체를 설명하는 단락이 거의 항상 등장한다. 이것이 고객사 보도자료와의 가장 큰 차이점이다.

**표준 매쉬 설명문:**
```
매쉬(mAsh)는 자체 개발한 딥러닝 기술을 바탕으로 매장 CCTV 영상을 정밀 분석하고, 방문객 동선을 끊김 없이 추적해 방문·구매 전환율 등 핵심성과지표(KPI)를 측정·모니터링하며, KPI 달성과 개선을 위한 방향성을 제안하는 솔루션이다.
```

**간결 버전:**
```
매쉬는 자체 딥러닝 기술을 기반으로 매장 CCTV 영상을 정밀 분석해 방문객 동선을 추적하고, 방문·구매 전환율 등 핵심성과지표(KPI)를 측정·모니터링하는 솔루션이다.
```

### 소제목으로 분리하는 패턴
일부 매체에서는 `### 매쉬(mAsh)란?` 소제목을 두고 별도 섹션으로 설명한다.

## 8. 문장 길이와 단락 구성

### 문장 길이
- **리드 문장:** 40~70자 (고객사 보도자료보다 약간 짧은 경향)
- **기능 설명 문장:** 40~80자 (기술 용어가 많아 길어질 수 있음)
- **마무리 인용문:** 60~100자 (직접 인용 포함)

### 단락 구성
- **단락당 1~3문장**
- 기능/특징 나열은 불릿 리스트로 정리하는 비율이 높음
- 소제목(###)으로 섹션 구분이 고객사 보도자료보다 더 빈번
- 일부 매체에서 핵심 요약 불릿을 리드 전에 배치

### 전형적인 기능 설명 불릿 리스트 형태
```
- **LLM 결합:** 기존 방문객 데이터 분석 기능에 거대 언어 모델을 결합한 대화형 AI 솔루션
- **사용 방법:** 웹 대시보드 '매쉬보드'에서 'AI 분석 기능' 클릭 → 지표 선택 → 질문 입력 → 응답 수신
- **보안:** AWS '아마존 베드록' 활용으로 엔터프라이즈급 보안 체계 적용
- **심층 분석:** 날씨·행사·상권 등 외부 변수와의 인과관계 분석
```

### 연결 표현
- `특히`, `또한`, `나아가`, `이를 통해`
- `향후`, `이번에 출시한`

## 10. 제품 보도자료 특유의 구성 요소

### 핵심 요약 블록 (일부 매체)
리드 앞에 3줄 불릿 요약을 배치하는 패턴:
```
- 매장 방문객 데이터 분석·요약해 운영 성과 개선 돕는 핵심 인사이트 도출
- 엔터프라이즈급 보안 체계, 강력한 암호화 정책으로 데이터 보호와 정보 오류 최소화
- 향후 데이터 자동 취합·분석하는 '자동 보고서 생성 기능' 추가 예정
```

### 사용 흐름(UI Flow) 설명
제품 보도자료에서 자주 등장하는 사용자 여정 서술:
```
사용자는 매쉬의 웹 대시보드 '매쉬보드(mAsh Board)'에서 'AI 분석 기능' 버튼을 클릭, 필요한 지표를 선택해 목적에 맞는 질문을 입력하면 정확한 응답을 얻을 수 있다.
```

### 기술 학술 인정 언급 (해당 시)
```
메이아이의 재식별 기술 성능을 분석한 논문 '사람 재식별 AI 모델의 카메라 편향에 대한 연구'는 세계 최고 권위 AI 학술대회 ICLR 2025에서 상위 5%에 부여되는 '스포트라이트'로 선정됐다.
```

## 11. 핵심 작성 원칙 요약

1. **리드에서 '무엇이 새로운지'를 즉시 전달한다:** 기존 제품(매쉬)에 무엇(매쉬 인사이트)이 추가되었는지를 첫 문장에 명시한다.
2. **기존 제품 맥락을 반드시 제공한다:** 신기능 설명 전에 매쉬가 무엇인지 1~2문장으로 설명하여 독자가 맥락을 이해할 수 있게 한다.
3. **기능 설명은 구체적으로, 사용 흐름까지 보여준다:** 버튼 클릭 → 지표 선택 → 질문 입력 → 응답 수신 등 실제 사용 과정을 서술한다.
4. **보안/인프라를 별도로 언급한다:** 엔터프라이즈 고객 대상이므로 AWS 베드록, 암호화, 민감 정보 보호 등을 반드시 포함한다.
5. **향후 로드맵으로 기대감을 만든다:** 자동 보고서 생성 등 다음 단계 기능을 구체적으로 예고한다.
6. **제품 담당자 코멘트로 마무리한다:** CEO가 아닌 제품팀 리드의 인용문으로, 기술적 깊이와 실무 관점을 전달한다.
7. **기술 용어는 첫 등장 시 풀네임을 병기한다:** `거대 언어 모델(LLM)`, `핵심성과지표(KPI)` 등 약어 사용 시 반드시 첫 등장에서 풀어 쓴다.

## 12. 고객사 보도자료 vs 제품 보도자료 주요 차이 비교

| 항목 | 고객사 도입 보도자료 | 제품 출시 보도자료 |
|------|----------------|---------------|
| 리드 핵심 정보 | 계약 체결 + 고객사명 + 매장 수 | 기능 추가 + 제품명 |
| 도입 경과 | PoC → 확장 타임라인 (필수) | 없음 |
| 기존 제품 설명 | 간략 (1~2문장) | 상세 (별도 섹션) |
| 신기능 UI 설명 | 없음 | 사용 흐름 서술 (필수) |
| 보안/인프라 | 개인정보 보호 정도 | AWS 베드록, 암호화 등 상세 |
| 향후 계획 | 매장 확장 계획 | 기능 로드맵 |
| 인용 주체 | CEO/CPO | 제품팀 리드 |
| 인용 내용 | 협업 의의 | 제품 가치 + 발전 방향 |
| 정확도 수치 | 90%, 98% 등 구체적 | 정성적 표현 위주 |
| 고객사 코멘트 | 있을 수 있음 | 없음 |
